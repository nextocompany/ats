## Deviations from Plan

1. **gosec failure unmasked (handled).** The plan scoped only golangci-lint + govulncheck because on `main` the `security` job failed at the *govulncheck* step (line 95–99), so GitHub Actions skipped the later *gosec* step (line 100–104) — its failure was invisible. Once govulncheck passed, gosec surfaced **3 pre-existing findings**, all in `cmd/importref/main.go`:
   - **G101** (LOW, line 29): localhost dev-fallback DSN string — the same credential already present in `docker-compose.yml`/`Makefile`/`ci.yml`; not a real secret (prod sets `DB_URL`).
   - **G304 + G703** (HIGH, line 59): `os.Open(path)` where `path` is an explicit `argv` CSV path to an operator-run admin import CLI — not untrusted external input.
   - **WHY annotate, not refactor**: this is a CI-fix PR; changing the CLI's runtime behavior (e.g. removing the dev DSN default, or scoping reads via `os.Root`) would break the tool's zero-config local UX and arbitrary-path imports. Justified `//#nosec Gxxx -- reason` is the idiomatic, behavior-preserving, transparent gosec resolution for admin tooling. **Flagged for reviewer scrutiny in `/code-review`.**

2. **Task 5 (`.golangci.yml`) skipped** — intentional. The plan marked it optional for drift-protection; pinning `version: v2.11.3` already freezes the linter set, so the config adds no value and only enlarges the diff.

## Issues Encountered

- **First `#nosec` attempt didn't register** (`Nosec: 0`, issues still 3). gosec requires the literal token `#nosec` (with hash) and is reliable only as a **same-line trailing** comment — my initial `// nosec` on a preceding line was ignored. Fixed by moving to `//#nosec Gxxx -- reason` trailing the flagged statement. Re-verified `Nosec` recognized, gosec exit 0.
