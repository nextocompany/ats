### MEDIUM

- **[M1] Exact-patch pins create a silent security-decay risk** — `.github/workflows/ci.yml` now pins `go-version: "1.26.4"` (×3) and `golangci-lint version: v2.11.3`. This is the intended determinism tradeoff (and is why the build is reproducible), but it means a *future* Go stdlib CVE fixed in 1.26.5+ will **not** be picked up automatically — the `security` job will keep passing against a now-vulnerable toolchain until someone manually bumps the pin. The govulncheck DB is always current, so it *would* flag a new CVE — but only once a contributor happens to bump Go past the float.
  - **Mitigation (folded into this PR):** added `.github/dependabot.yml` with weekly `github-actions` + backend `gomod` updates. ⚠️ Honest limitation documented in that file — Dependabot bumps `require` deps (future `x/net`-style CVEs) and `uses:` action refs, but does **not** bump the `go`/`toolchain` directive or the free-form `go-version:` workflow input, so the pinned Go patch still needs a periodic manual review. Partial mitigation; residual is process, not blocking.

### LOW

- **[L1] Hardcoded dev DSN remains in source** — `backend/cmd/importref/main.go:29` keeps `postgres://hruser:hrpass@localhost:5432/hr_db?sslmode=disable` as a fallback, now `//#nosec G101`. Acceptable: it is the well-known local-dev credential already duplicated in `docker-compose.yml`, `Makefile`, and `ci.yml` env, it is dev-only (prod sets `DB_URL`), and the suppression carries a clear justification. Optional cleaner posture (future): drop the fallback and require `DB_URL`, so no credential string lives in source at all. Not blocking — behavior change out of scope for a CI fix.

- **[L2] `#nosec G304,G703` on the CSV reader** — `backend/cmd/importref/main.go:59`. Justification is sound: `importref` is an operator-run admin CLI and `path` is an explicit `argv` argument, not untrusted external input. ✔️ Correctly scoped to the single `os.Open` line (not a file- or function-wide suppression), and the rule IDs are named explicitly rather than a blanket `#nosec`. This is the idiomatic, transparent resolution for admin tooling.

- **[L3] `go.mod` raised to `go 1.26.4` with no separate `toolchain` directive** — informational. With `GOTOOLCHAIN=auto` (the repo default), the `go 1.26.4` directive alone triggers the 1.26.4 toolchain download for local devs; no action needed. Confirmed locally (`go version` auto-switched to 1.26.4).
