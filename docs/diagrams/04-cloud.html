CI/CD และ Secrets

Pipeline (.github/workflows/deploy.yml)

  • Trigger: workflow_dispatch หรือ push tag v*
  • Auth เข้า Azure ด้วย OIDC federated (ไม่มี secret ค้าง)
  • Build ฝั่ง server: az acr build (ไม่ใช้ Docker บน runner) · backend loop สร้าง api/worker/scheduler · web build portal/dashboard พร้อม baked NEXT_PUBLIC_*
  • az containerapp update --image ...:TAG ทั้ง 5 แอป แล้ว smoke check /health
  • ไม่รัน DB migration อัตโนมัติ — ทำมือด้วย golang-migrate

การจัดการ secret

  • โหมด RBAC (ค่าเริ่มต้น): secret อยู่ใน Key Vault · Container App อ้างผ่าน keyVaultUrl + UAMI · env ใช้ secretRef
  • เว็บแอปไม่มี secret runtime — ค่าฝังตอน build

โดเมน: <app>.yellowmoss-b9b985f7.southeastasia.azurecontainerapps.io

เส้นทาง traffic: Internet → ACA ingress (TLS) → portal/dashboard + api → API enqueue Redis → worker/scheduler บริโภค → ทุก service เข้าถึง Postgres/Blob/Redis + AI plane (eastus)

หมายเหตุที่ควรรู้ — แม้ comment ใน Bicep จะเขียนว่า gpt-4o แต่ค่า default จริงคือ gpt-4o-mini · และ deployment embeddings (text-embedding-3-small) ถูกใช้โดยแอป แต่ไม่ได้ประกาศไว้ใน Bicep module ของ OpenAI ต้องสร้างแยกเองก่อนเปิด hybrid search · Postgres/Redis/Storage ฯลฯ เปิด public network ทั้งหมด (v1 ยังไม่มี VNet)
