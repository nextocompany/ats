คำอธิบาย

SSO = พิสูจน์ว่าเป็นใคร, in-app = บอกว่าทำอะไรได้ — Entra ทำหน้าที่ authentication เท่านั้น ส่วนสิทธิ์ (authorization) มาจาก user store ของแอปเอง admin กำหนด role + scope + DPO ต่อบัญชีได้ ทำให้ผู้ใช้ SSO ที่ยังไม่มี role จะเจอ default-deny

fail-static ไม่ fail-open — ถ้าโหลดเมทริกซ์ RBAC จาก DB ไม่ได้ จะใช้ตารางสำรองที่ compile มากับโค้ด ไม่ใช่ปล่อยผ่านทุกอย่าง

permission catalog — เช่น executive.view, reports.export, bulk.upload, offer.write, onboarding.write, approval.decide_l1..l4, requisition.manage, pdpa.admin, users.admin · super_admin เป็น bypass ในโค้ดถือสิทธิ์ทั้งหมด

2 cookie แยกกัน — cp_session (ผู้สมัคร) และ hr_auth (HR password) ไม่ปะปนกัน · dashboard ยังมี marker cookie hr_session = entra/pw/dev สำหรับ gate route
