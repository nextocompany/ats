สิ่งที่ต้องสร้างใหม่ (gap จากระบบปัจจุบัน)
โครงสร้างใหม่
  • Area — ตาราง areas + area_stores (M:N) + user_areas + admin UI (ปัจจุบันมีแค่ subregion hardcode 13 ค่า)
  • Requisition link — vacancies.hiring_manager_user_id (FK จริง) + applications.vacancy_id (ปัจจุบันขาด link นี้)
  • Lock — ตาราง candidate_locks + TTL (ปัจจุบันไม่มีเลย)
  • Timer — applications.picked_up_at + release sweep (ลอก pattern approval:sla_sweep)
  • scope kinds — เพิ่ม area, requisition ใน CHECK
มีฐานรองรับแล้ว
  • dynamic RBAC รองรับ scope_kind ต่อ role (แค่ขยาย CHECK)
  • central pool มีฐาน: talent_pool + assigned_store_id=NULL + candidate_accounts
  • timeout sweep ลอก pattern approval:sla_sweep (index status,due_at) ได้
  • resolve actor จาก email (hrauth.FindByEmail) ใช้ผูก hiring manager ได้

ประมาณการ ~55-65 md แบ่ง 3 sub-phase (core → pool+lock → frontend+approval)
