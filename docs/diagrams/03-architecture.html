03 · Application Architecture

สถาปัตยกรรมแอปพลิเคชัน

โมดูลภายใน (internal packages) จัดกลุ่มตามโดเมน
Pipeline & AI คัดกรอง
  • pipeline — task process_application รัน pipeline เต็ม
  • ai — OCR + parse + embeddings (mock/azure/gemini)
  • scoring — ให้คะแนน hybrid ถ่วงน้ำหนักต่อตำแหน่ง + must-have gate
  • dedup — รวมผู้สมัครซ้ำเป็น canonical
  • branch — จังหวัด→subregion→สาขา + คะแนนทำเล
  • fit — วิเคราะห์ความเหมาะข้ามตำแหน่ง (sync)
  • interview — สัมภาษณ์ AI แบบ adaptive (sync)
  • search — seam ค้นหา (trigram / AI Search)
โดเมนใบสมัคร / HR workflow
  • applications — แกนกลาง: persistence, status machine, schedule, feedback, approval, offer, letters, onboarding, bulk
  • candidates / profiles — ข้อมูล + read API ของ HR
  • intake — webhook รับเข้า (HMAC) → pipeline
  • positions / vacancies / requisitions — JD, ตำแหน่งว่าง, การเปิดรับ
  • stores — ข้อมูลสาขา + แมป subregion
  • letters — เรนเดอร์จดหมาย PDF (ไทย)
  • calendar — นัดสัมภาษณ์/Teams ผ่าน Graph
  • reengage — ติดต่อ talent pool กลับ
ตัวตน / สิทธิ์ / การแจ้งเตือน
  • candidateauth — บัญชีผู้สมัคร (OTP/LINE/Google), resume library, consent
  • lineauth / auth — LINE OAuth + verify id-token
  • hrauth — password sign-in ของ HR
  • rbac / rbacadmin — role→permission→scope (DB-backed, fail-static)
  • middleware — auth, logging, client-IP, CSRF/origin guard
  • notify — seam แจ้งเตือน (LINE/อีเมล/Teams)
  • members / users / settings — บัญชี, /me, config runtime
PDPA & การกำกับดูแล
  • pdpa / pdpaadmin — consent, retention/erasure, คอนโซล DPO
  • dsar — คำขอเข้าถึง/พกพา/ลบข้อมูล (ผู้สมัคร)
  • breach — ทะเบียนเหตุข้อมูลรั่ว + นับถอยหลัง 72 ชม.
  • activity — audit log เส้นทางผู้สมัคร
รายงาน & ภายนอก
  • reports — funnel, KPI, ส่งออก CSV
  • executive — overview ผู้บริหาร (budget vs headcount)
  • peoplesoft — รับ webhook เข้า + sync ออก (Integration Broker)
  • public — API สาธารณะของ portal (apply, status)
  • health — รวม probe ของ dependency เป็น /health
หลักการออกแบบ

codebase เดียว 3 entrypoint — api/worker/scheduler build จาก backend/Dockerfile ตัวเดียวด้วย build-arg SVC ทำให้ใช้โค้ดร่วมกันแต่แยก image

provider seam — ทุกบริการภายนอกซ่อนหลัง interface + factory ที่เลือก mock/real ตาม env ทดสอบได้โดยไม่ต้องมีของจริง
