CP AXTRAAI HR ATS · System Diagrams
เอกสารสถาปัตยกรรมระบบ

ระบบ AI คัดกรองและจัดการผู้สมัครงาน

ชุดแผนผังอธิบายระบบ ATS (Applicant Tracking System) ของ CP Axtra ทั้งหมด ตั้งแต่ภาพรวม เส้นทางผู้สมัครแบบ end-to-end สถาปัตยกรรมแอปพลิเคชัน โครงสร้างบนคลาวด์ ระบบยืนยันตัวตน/สิทธิ์ ไปจนถึงการเชื่อมต่อภายนอกและการแจ้งเตือน ทุกแผนผังอ้างอิงจากซอร์สโค้ดจริง

01 · OVERALL
ภาพรวมระบบ (System Context)
ใครใช้ระบบ ระบบประกอบด้วยอะไร และเชื่อมต่อกับบริการภายนอกใดบ้าง มองจากภายนอกเข้ามา
02 · END-TO-END
เส้นทางผู้สมัคร (End-to-End Flow)
ตั้งแต่ยื่นใบสมัคร → OCR/แยกข้อมูล → ตรวจซ้ำ → ให้คะแนน AI → สัมภาษณ์ → อนุมัติ → จ้างงาน พร้อม status machine
03 · ARCHITECTURE
สถาปัตยกรรมแอปพลิเคชัน
Frontend 2 แอป, Go API / Worker / Scheduler, คิวงานแบบ async, ชั้นข้อมูล และโมดูลภายใน 40+ แพ็กเกจ
04 · CLOUD
โครงสร้างบน Azure (Cloud)
Azure Container Apps, PostgreSQL, Redis, Blob, OpenAI, AI Search, Key Vault และเส้นทาง traffic
05 · AUTH & RBAC
ยืนยันตัวตนและสิทธิ์การเข้าถึง
2 ระนาบตัวตน (ผู้สมัคร vs HR), Entra SSO / LINE / Google / OTP และโมเดล RBAC แบบ scope ตามสาขา/ภูมิภาค
06 · INTEGRATIONS
การเชื่อมต่อภายนอกและการแจ้งเตือน
Azure AI, LINE, Microsoft Graph/Teams, อีเมล ACS, PeopleSoft HCM และระบบสลับ mock/real ทุกตัว
07 · PRICING
ประเมินราคา 3 Phase
แบ่ง Pool / Portal / ATS · ค่าพัฒนาคิดจาก man-day × rate (กรอกเอง) + ค่าคลาวด์รายเดือน คำนวณสดในหน้า
08 · RBAC REDESIGN
โมเดลสิทธิ์ใหม่ (Proposed)
2 แกน (มองเห็น × ทำได้), 7 roles, area แบบ dynamic, candidate pool + lock + timeout 3 วัน, แผน migrate role เดิม
สรุประบบโดยย่อ

ระบบทำอะไร — รับใบสมัครจากหลายช่องทาง (career portal, อัปโหลดเป็นชุด, ฟอร์มภายนอก, PeopleSoft) แล้วใช้ AI อ่านเรซูเม่ (OCR), แยกข้อมูล, ตรวจผู้สมัครซ้ำ, ให้คะแนนความเหมาะสมตามตำแหน่ง, สัมภาษณ์เบื้องต้นด้วย AI, จัดสาขา/talent pool และส่งต่อให้ HR ทำขั้นตอนสัมภาษณ์ อนุมัติหลายระดับ ออกข้อเสนอ และ onboarding จนถึงการ sync กลับ PeopleSoft เมื่อจ้างงาน
